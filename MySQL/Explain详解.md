# explain详解

## explain作用

可以查看SQL语句的具体执行过程，

例子：`explain select ...`



## explain执行计划包括

### 1.id：查询序列号

`select`查询的序列号(一组数字)，表示查询中执行`select`子句或者操作表的顺序。

包含三种情况：

1、如果`id`相同，那么执行顺序从上到下

2、如果`id`不同，如果是子查询，`id`的序号会递增，id值越大优先级越高，越先被执行

3、`id`相同和不同的，同时存在：相同的可以认为是一组，从上往下顺序执行，在所有组中，id值越大，优先级越高，越先执行
